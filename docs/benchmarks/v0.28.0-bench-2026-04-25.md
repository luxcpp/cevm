# cevm v0.28.0 — GPU-vs-CPU benchmark sweep

Apple M1 Max, 10-core CPU, 32-core GPU, 64GB unified memory.

## Three perf landmines in v0.27, three fixes in v0.28

1. **Per-tx scratch sized 64 KiB unconditionally** → switched to per-tx
   static bytecode analysis (`scan_max_memory`, `batch_max_memory` in
   `lib/evm/gpu/kernel/evm_kernel_host.mm`). Typical AMM swap drops from
   64 KiB to ~256 bytes.
2. **`waitUntilCompleted` per dispatch + fresh `EvmKernelHost` per call**
   → cached process-singleton host in `gpu_dispatch.cpp` so the Metal
   pipeline state object is reused instead of recompiled on every call.
   Parity test runtime fell from 10.1 s → 0.115 s as a side-effect (88×).
3. **Per-tx memory cap not threaded into the kernel** → kernel now
   takes `mem_cap` per tx (`offset_in_bounds`, `expand_mem_words`,
   `expand_mem_range` in `evm_kernel.metal`) so the host's tighter
   bound is enforced as OOG inside the kernel rather than silent OOB
   into the next tx's slot.

## 1M-tx maximum-contention demo (amm_swap, all txs touch reserve0/1)

```
N          | CPU-Seq (ms)  | CPU-Par (ms)  | GPU-Met (ms)  | par/seq | gpu/par
-----------|---------------|---------------|---------------|---------|--------
100,000    |       206.12  |       179.32  |        509.06 |   1.15× |  0.35×
500,000    |      1068.51  |       815.25  |        713.55 |   1.31× |  1.14×
1,000,000  |      2167.66  |      1796.32  |       1225.04 |   1.21× |  1.47× ★
```

v0.27 OOM'd at 1M (`Metal buffer allocation failed` — 64 KiB × 1M =
64 GiB scratch). v0.28 runs cleanly and **GPU beats CPU-Par by 1.47× at
1M** even on the worst-case 100% write-write contention workload.

## What's left for v0.29 (V3 persistent kernel)

1. Persistent kernels with lock-free queues — eliminate per-launch
   dispatch latency entirely. Targeted speedup: another 4–8×.
2. SIMD-cooperative interpreter (32 threads/tx on hot opcodes:
   keccak, ecrecover, modexp). Targeted: 8–16× on crypto-heavy.
3. Wave/FPC pipelining — start wave[t+1] before wave[t] commits.
   Targeted: ~90% device utilisation steady state.

These together get to the 10–100× target. v0.28 is the foundation;
v0.29 is the win.
